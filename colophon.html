How this site is built

Plain HTML and CSS. No framework. No build step. Under 40 KB total.

The whole site is a folder of static files. There’s no Next.js, no Gatsby, no Webpack, no Tailwind, no Node runtime involved at serve time. I could serve it with python -m http.server and it would work. It’s hosted on GitHub Pages out of the buik1/devops-portfolio-website repo.

Typography

  • Body: system sans stack (-apple-system, Segoe UI, Inter, Helvetica).
  • Headings and lede: Charter, falling back to Iowan Old Style, Palatino, Georgia.
  • Code: JetBrains Mono if you have it, otherwise SF Mono, Menlo, Consolas.

No web fonts. The site loads with whatever you already have, which means it’s instantaneous and it respects your system.

Design choices

  • One accent colour (a navy that prints well).
  • Text maxes out at a reading-friendly ~68 characters per line.
  • Every case study is one long scrollable page. I don’t hide writing behind tabs.
  • SVG diagrams are hand-written, not exported from draw.io. They’re small and they scale.
  • Dark mode is on the list. I haven’t done it yet because I’d rather publish than polish.

Why no framework

Because I wanted to prove to myself I could still build a site without one. Also because I keep writing long-form notes about infrastructure, and the lifecycle of a blog post is about ten years longer than the lifecycle of any JS framework I might pick today.
